Unit Tests - Good

  1. Fast
  2. Simple
  3. Easy to read
  4. Easy to understand
  5. Tests one and only one functionality

Unit Tests - Bad

  1. Slow
  2. Complex
  3. Hard to read
  4. Hard to understand
  5. Tests multiple things
  6. Complex branching and looping in tests

Please write Good Unit Tests!

Conventions

  1. Put your Unit Test files ( aka Spec ) files as close as possible to what you are testing
  2. Follow the naming convention [module/File]Spec.js
  3. Follow the folder structure proposed: by feature not by type
